# Playbook Doctrine

This file is hand-curated. Use it for durable rules of engagement that should shape agent synthesis before any live or historical tool output is interpreted.

## Universal Rules

- A key level is not a setup. Treat levels as attention zones until flow confirms.
- Every setup read should distinguish: location, trigger, confirmation, invalidation, and risk state.
- When structure and flow conflict, report the conflict. Do not force a single answer.
- Report sample size and reliability tier with any edge claim.

## Continuation Families

- Best when the session is proving one-sided acceptance away from balance.
- Weak when IB extension becomes two-sided, RVOL is low, or price repeatedly re-enters value.
- Do not treat first break as enough. Prefer pullback/retest behavior with participation confirmation.

## Responsive / Repair Families

- Best when the session is migrating, repairing inventory, or rejecting away from value.
- Weak when single prints and one-sided value migration remain intact.
- Prefer clear destination framing: VWAP, POC, VA edge, DNP, or prior value reference.

## Failure / Vacuum Families

- Resting size alone is not enough.
- Require failed defense, pace expansion, and liquidity pull or inability to refill.
- Invalidation belongs back inside the failed defense zone.
